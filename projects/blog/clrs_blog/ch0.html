Peak Finder

問題敘述

在一個 1 維 (1D) 的陣列中，我們希望找到一個局部最高點 Peak。

相關定義

上述描述並不是非常清楚，下方我們給出了關於本問題中相關物件較為嚴格的定義，以便能夠精確描述這個問題。

  • 正整數的記號：$$\mathbb{N} := \{1, 2, \dots\}$$
  • \(1\sim n\) 的正整數集合：$$\mathbb{N}_{\le n} := \{1, 2, \dots, n\}, n\in \mathbb{N}$$
  • 長度為 \(n\) 的 1D 陣列： $$A_n := \{a_i\in \mathbb{N}: i \in \mathbb{N}_{\le n}\}, n\in \mathbb{N}$$ 下方的討論除非有特別指出，否則 \(A_n\) 默認為某個長度為 \(n\) 的 1D 陣列。
  • \(A_n\) 中元素 \(a_i\) 的鄰居：$$\text{nbr}(a_i) := \{ a_j\in A_n: |i-j| = 1 \}$$
  • \(A_n\) 的局部最高點 Peak：$$\text{Peak}(A_n) := \{a_i\in A_n:\forall x\in \text{nbr}(a_i),a_i \ge x\}$$

上述問題可以重新描述為：對於長度為 \(n\) 的 1D 陣列 \(A_n\) ，找出 \(a\in \text{Peak}(A_n)\)

簡單觀察

  • 可以發現 \(A_n\) 的 Peak 並保證不唯一，即 \(\exists n\in \mathbb{N},\exists A_n:\text{Peak}(A_n) \ge 2\)
  • \(A_n\) 的 Peak 是否一定存在？
    • 定義 \(A_n\) 最大值為 $$\max(A_n):=\{x\in A_n: \forall y\in A_n, x \ge y\}$$
    • 觀察到 \(A_n\) 最大值一定是 Peak：\(\max(A_n)\subseteq \text{Peak}(A_n)\)
    • 而 \(A_n\) 最大值一定存在，具體證明可由反證法或是良序原理推得。

直覺解法

從某一端開始，依序逐個元素檢查是否滿足條件，直到抵達另一端。當抵達另一端時，該元素必為 Peak。

分治解法

分治 Divide and Conquer，字面意思為「分而治之」，故名「分治法」。其思想為將問題分解成數個相似的子問題，藉由解決這些子問題來得到原問題的答案，通常我們期望這樣能夠降低問題的複雜度並減少解決時間。

我們可以挑選 \(A_n\) 中的位於中央的元素 \(a_{\lceil n/2\rceil}=a_m\)，並對其進行如下檢查：

  • 如果 \(a_{m-1}\) 存在且 \(a_{m-1} > a_m\) ，則僅尋找新 1D 陣列 \(L=\{a_1,\dots,a_{m-1}\}\) 中的 Peak
  • 如果 \(a_{m+1}\) 存在且 \(a_{m+1} > a_m\) ，則僅尋找新 1D 陣列 \(R=\{a_{m+1},\dots,a_{n}\}\) 中的 Peak
  • 若上述條件皆不成立，則 \(a_{m}\in \text{Peak}(A_n)\)

該演算法的時間複雜度

時間複雜度 Time Complexity，一種用於描述或評估演算法從開始到結束的所需時間。一般來說，每個演算法需要關注的有：
  • 平均情況下的時間複雜度
  • 最差情況下的時間複雜度
  • 最好情況下的時間複雜度
其中又以平均的時間複雜度最為重要，其次為最差時間複雜度。我們會使用 \(o,O,\Theta,\Omega,\omega\) 符號來表示時間複雜度
  • \(o(n^4)\) 表示該演算法需要時間 遠低於 \(n^4\) 級別，可以類比為遠小於符號 \(\ll\)
  • \(O(n^4)\) 表示該演算法需要時間 不高於 \(n^4\) 級別，可以類比為小於等於符號 \(\le\)
  • \(\Theta(n^4)\) 表示該演算法需要時間 約等於 \(n^4\) 級別，可以類比為等於符號 \(=\)
  • \(\Omega(n^4)\) 表示該演算法需要時間 不低於 \(n^4\) 級別，可以類比為大於等於符號 \(\ge\)
  • \(\omega(n^4)\) 表示該演算法需要時間 遠高於 \(n^4\) 級別，可以類比為遠大於符號 \(\gg\)
我們一般會使用 \(T(n)\) 來描述演算法的時間複雜度，例如 \(T(n)=O(n^3), T(n)=\omega(n\lg n + n)\)
這裡的 \(n\) 表示問題的規模，以本問題為例，問題規模即為 1D 陣列的大小 \(|A_n|=n\)。

由於挑選中央元素並比較的這個動作視為常數時間，該演算法的時間複雜度可以表示為 $$T(n)=T(n/2)+\Theta(1)$$ 這個式子可以一直寫下去 $$T(n)=T(n/2)+\Theta(1)=\dots=T(n/2^i)+i\times\Theta(1)$$ 當 \(T(n/2^i)\) 足夠小，可以將其視為常數時間，於是可得 $$T(n)=T(n/2^i)+i\times\Theta(1)=\dots=\lg(n)\times\Theta(1)=\Theta(\lg(n))$$ 由於問題規模是有限的，由阿基米德性質，我們總是可以保證能夠使得 \(T(n/2^i)\) 足夠小而視為常數。

2D Peak Finder

將原來的 1D 陣列改為 2D 陣列，同樣尋找該 2D 陣列的 Peak。

相關定義

  • 2D 陣列：$$M_{mn} := \{a_{ij}\in \mathbb{N}_0: (i,j) \in X\times Y \subset \mathbb{N}_{\le m}\times\mathbb{N}_{\le n}\}, m,n\in \mathbb{N}$$
  • \(M_{mn}\) 中元素 \(a_{ij}\) 的鄰居：$$\text{nbr}(a_{ij}) := \{ a_{i'j'}\in A: |i-i'| + |j-j'| = 1 \}$$
  • \(M_{mn}\) 的局部最高點 Peak：$$\text{Peak}(M_{mn}) := \{a_{ij}\in M_{mn}:\forall x\in \text{nbr}(a_{ij}),a_{ij} \ge x\}$$

上述問題可以重新描述為，對於大小為 \(m\times n\) 的 2D 陣列 \(M_{mn}\) ，找出 \(a\in \text{Peak}(M_{mn})\)
同時，為了避免混淆，我們將前述 1D 陣列的 Peak 稱為 1D-Peak。

嘗試解法 #1

  • 找到 \(M_{mn}\) 中間行 (middle column) ，即第 \(j=m/2\) 行的 1D-Peak
  • 找到 \(M_{mn}\) 第 \(j\) 列 (\(j\)th row) 的 1D-Peak
  • 上一步找到的 1D-Peak 即為 \(M_{mn}\) 的 Peak

該解法不正確，可找到會給出錯誤答案的例子。

嘗試解法 #2

  • 找到 \(M_{mn}\) 中間行 (middle column) ，即第 \(j=m/2\) 行的 \(\max\)，假設為 \(a_{ij}\)
  • 對於 \(M_{mn}\) 第 \(j\) 列 (\(j\)th row) ，比較 \(a_{ij}\) 的鄰居
    • 對於鄰居 \(a_{ik}\) ，如果 \(a_{ik} > a_{ij}\) ，則選擇該側所有的行，即 \(1\sim k\) 或是 \(k \sim n\)，並對這些行尋找 Peak
    • 如果 \(a_{ij}\) 皆不嚴格大於其鄰居，則 \(a_{ij}\) 就是所尋找的 Peak
  • 如果僅剩下一行，則該行的 \(\max\) 就是所尋找的 Peak

該演算法的時間複雜度

由於挑選最大值並比較的這個動作與 \(m\) 成線性關係，即 \(\Theta(m)\)，該演算法的時間複雜度可以表示為 $$T(m, n)=T(m, n/2)+\Theta(m)$$ 理由與前述類似，可以將其寫為 $$T(m, n)=T(m, n/2^i)+i\times\Theta(m)=\dots=\lg(n)\times\Theta(m)=\Theta(m\lg(n))$$
